# Signals

A signal is a custom text label that identifies the electrical signal carried by a conductor. For example `VCC` or `CAN H`.

Signals can be entered into the cavity of a connector or terminal. When a signal is assigned to a cavity, it automatically propagates to all other cavities/ terminals on the same [Net](/feature/nets), eliminating the need to manually enter the same label at every connection point.

## <IconBolt/> Propogate Signal

This is the default when text is entered. This will autopopulate any empty cavity on the same net.
<br/>
<video src="/feature/signals-normal.mp4" width={800} style={{ borderRadius: '8px' }} autoPlay loop muted playsInline />

## <IconBoltOff/> Do not Propogate Signal

Right click **Signal** and select <IconBoltOff/> `Do not propogate`. Signal will no longer propogate through out the net.
<br/>
<video src="/feature/signals-propagate-no.mp4" width={800} style={{ borderRadius: '8px' }} autoPlay loop muted playsInline />


## <IconWorldBolt/> Global Signal

 Right click **Signal** and select <IconWorldBolt/> `Make signal global`. If two **Signals** have the same text label and are <IconWorldBolt/> `Global` they will be seen as connected and on the same net.
<br/>
<video src="/feature/signals-global.mp4" width={800} style={{ borderRadius: '8px' }} autoPlay loop muted playsInline />
 
## Signal Conflicts

If there are more than one signal on the same net, empty cavities will show all signals connected on that net.
